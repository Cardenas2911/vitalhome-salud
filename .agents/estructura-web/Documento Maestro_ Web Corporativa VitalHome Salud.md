# Optimización de Conversión (CRO) y Marcado Schema para la Web Corporativa de VitalHome Salud

Como Director de Estrategia Digital y Arquitecto de Información, he desarrollado las recomendaciones de Optimización de Conversión (CRO) y el marcado Schema.org específico para cada sección de la Web Corporativa de VitalHome Salud. El objetivo es maximizar la visibilidad en motores de búsqueda (AEO/IA) y guiar al usuario hacia la acción, proyectando una imagen de solvencia y autoridad.

---

## Secciones de la Home Page (o Páginas de Servicio Clave)

### 1. HERO CORPORATIVO (Propuesta de Valor Institucional)

*   **Recomendaciones CRO:**
    *   **CTA Principal:** Visible y contrastado, con micro-copy que resalte el beneficio (`Agendar Consultoría Exclusiva`).
    *   **Sub-CTA (Opcional):** Para usuarios menos comprometidos (`Descargar Guía Gratuita de Cuidado`).
    *   **Video Corto (90s):** Explicativo de la propuesta de valor, con subtítulos y opción de transcripción.
    *   **Indicador de Confianza:** Pequeño logo de "+X años de experiencia" o "+Y familias atendidas".
*   **Marcado Schema Avanzado:**
    *   **Tipo JSON-LD:** `Organization`, `WebSite`.
    *   **Propiedades Clave:** `name`, `url`, `logo`, `description`, `contactPoint`, `sameAs`, `potentialAction` (para la barra de búsqueda del sitio), `hasOffer` (si el CTA principal es una oferta de servicio).

### 2. UNIDADES DE NEGOCIO (Soluciones Estratégicas)

*   **Recomendaciones CRO:**
    *   **CTAs Secundarios:** Un CTA claro por cada unidad de negocio (`Explorar Enfermería Domiciliaria`, `Ver Planes de Rehabilitación`).
    *   **Iconografía:** Clara y representativa para cada servicio.
    *   **"Quick View" o Tooltip:** Al pasar el cursor sobre un servicio, mostrar un breve resumen de beneficios.
    *   **Anclas:** Enlaces internos que lleven a secciones más detalladas de cada servicio.
*   **Marcado Schema Avanzado:**
    *   **Tipo JSON-LD:** `Service` (para cada unidad de negocio, anidado bajo `Organization`).
    *   **Propiedades Clave:** `name`, `description`, `serviceType`, `areaServed`, `provider` (VitalHome Salud), `offers` (precio o rango de precios si aplica, o indicación de "consultar").

### 3. MÉTRICAS DE IMPACTO (El "Antes y Después" Institucional)

*   **Recomendaciones CRO:**
    *   **Visualización de Datos:** Gráficos interactivos que permitan al usuario filtrar o ver detalles.
    *   **CTA:** `Ver Casos de Éxito Completos` o `Descargar Informe de Impacto`.
    *   **Micro-copy de Urgencia/Escasez (si aplica):** "Solo X cupos disponibles para evaluación gratuita este mes".
*   **Marcado Schema Avanzado:**
    *   **Tipo JSON-LD:** `AggregateRating` o `Review` (si se asocia a testimonios de impacto), o `WebContent` con `about` propiedades que describan los resultados.
    *   **Propiedades Clave:** `itemReviewed` (VitalHome Salud), `ratingValue`, `reviewCount` (si aplica), `headline` (para el contenido).

### 4. ADN / METODOLOGÍA PROPIETARIA

*   **Recomendaciones CRO:**
    *   **CTA:** `Conocer Metodología VitalCare® a Fondo` (llevar a página dedicada).
    *   **Infografía Descargable:** Ofrecer una versión en PDF de la metodología para profundizar.
    *   **Video Explicativo:** Animación corta que explique el proceso de la metodología.
*   **Marcado Schema Avanzado:**
    *   **Tipo JSON-LD:** `HowTo` o `CreativeWork` (con `about` para describir la metodología).
    *   **Propiedades Clave:** `name`, `description`, `step` (para `HowTo`), `author` (VitalHome Salud).

### 5. EL VIAJE DEL CLIENTE (Proceso)

*   **Recomendaciones CRO:**
    *   **CTA:** `Iniciar su Proceso de Cuidado Ahora` (llevar a formulario de contacto o evaluación inicial).
    *   **Chatbot/Asistente Virtual:** Para guiar al usuario a través del proceso y responder dudas frecuentes.
    *   **Testimonios Cortos:** Integrar citas de clientes satisfechos en cada etapa del proceso.
*   **Marcado Schema Avanzado:**
    *   **Tipo JSON-LD:** `Process` (si existe un tipo específico para procesos, o `HowTo` si se puede adaptar).
    *   **Propiedades Clave:** `name`, `description`, `step` (con `name` y `text` para cada fase).

### 6. INFRAESTRUCTURA TÉCNICA (Diferenciadores)

*   **Recomendaciones CRO:**
    *   **CTA:** `Descubrir Nuestra Tecnología en Detalle` (llevar a página dedicada).
    *   **Demo Interactiva:** Pequeña simulación de la plataforma de monitoreo.
    *   **Certificaciones de Seguridad:** Logos de certificaciones de seguridad de datos (ej. ISO 27001, HIPAA compliance).
*   **Marcado Schema Avanzado:**
    *   **Tipo JSON-LD:** `TechArticle` o `AboutPage` (con `mainEntity` describiendo la tecnología).
    *   **Propiedades Clave:** `headline`, `articleBody` (describiendo el stack), `keywords` (headless, monitoreo remoto, seguridad de datos), `author` (VitalHome Salud).

### 7. MATRIZ DE DIFERENCIACIÓN (Tabla Comparativa)

*   **Recomendaciones CRO:**
    *   **CTA:** `Comparar Nuestro Enfoque y Solicitar Evaluación`.
    *   **"Generador de Comparación":** Herramienta interactiva donde el usuario selecciona opciones y ve la diferencia.
    *   **Testimonios Específicos:** Citas que resalten un punto de diferenciación clave.
*   **Marcado Schema Avanzado:**
    *   **Tipo JSON-LD:** `Table` (si existe un tipo específico para tablas comparativas, o `WebContent` con `about` propiedades que describan la comparación).
    *   **Propiedades Clave:** `name`, `description`, `hasPart` (para cada elemento comparado).

### 8. VALIDACIÓN INSTITUCIONAL (Social Proof)

*   **Recomendaciones CRO:**
    *   **CTA:** `Ver Todos los Testimonios` o `Conocer Nuestras Certificaciones`.
    *   **Video Testimonios:** Cortos videos de clientes C-Level.
    *   **Logos de Aliados:** Interactivos, al hacer clic, llevan a una página de alianzas o descripción.
*   **Marcado Schema Avanzado:**
    *   **Tipo JSON-LD:** `Review` y `Organization` (con `aggregateRating` y `review` propiedades).
    *   **Propiedades Clave:** `itemReviewed` (VitalHome Salud), `author`, `reviewRating`, `reviewBody` (para testimonios), `publisher` (para certificaciones).

### 9. LEAD MAGNET ESTRATÉGICO (Conversión de Valor)

*   **Recomendaciones CRO:**
    *   **CTA Principal:** `Solicitar Consultoría Gratuita Ahora` (botón grande, color contrastante).
    *   **Formulario Corto:** Pedir solo la información esencial (nombre, email, teléfono).
    *   **Beneficios Claros:** Listar 3-5 beneficios clave de la consultoría/auditoría.
    *   **Prueba Social:** "Más de X familias ya han solicitado su consultoría".
*   **Marcado Schema Avanzado:**
    *   **Tipo JSON-LD:** `Offer` o `Service` (con `offers` describiendo la consultoría gratuita).
    *   **Propiedades Clave:** `name`, `description`, `price` (0 o "Gratis"), `priceCurrency`, `areaServed`, `provider`.

### 10. HUB DE CONOCIMIENTO (FAQ + Recursos)

*   **Recomendaciones CRO:**
    *   **CTA:** `Explorar Todos los Recursos` o `Ir al Blog`.
    *   **Barra de Búsqueda:** Funcional para el contenido del Hub.
    *   **Preguntas Frecuentes (FAQ):** Implementar un acordeón interactivo.
    *   **Contenido Destacado:** Mostrar los 3-5 artículos más populares o recientes.
*   **Marcado Schema Avanzado:**
    *   **Tipo JSON-LD:** `FAQPage` y `WebPage` (con `mainEntity` para los recursos).
    *   **Propiedades Clave:** `mainEntity` (para `FAQPage`, con `Question` y `Answer`), `name`, `description`, `hasPart` (para los recursos, con `CreativeWork` o `Article`).
